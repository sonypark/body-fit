# 체형 교정 & 맞춤 운동 추천 앱 — 기획/설계서

> 상태: **v0.2 (작동 프로토타입 반영)** · 대상 사용자: 학생 · 플랫폼: Web 우선 → App 확장 · 디자인 톤: 밝고 캐주얼한 게임형 UI
>
> 최근 업데이트(v0.2): 실제 유튜브 영상 연동, EXP 쿨다운 규칙, 완료/미완료 버튼 구분, 프로필 수정, 추천 랭킹+더보기(3→9), 로컬→원격 실행 전략 반영. 구현 현황은 **부록 A** 참조. 프로토타입 파일: `index.html`.

## 1. 개요

### 1.1 한 줄 정의
키·몸무게와 목표를 입력하면 **체형(BMI)·목표·운동 부위**에 맞는 운동/스트레칭을 짧은 클립과 함께 추천하고, 완료할 때마다 **경험치와 캐릭터 성장**으로 꾸준함을 보상하는 학생용 운동 코치 앱.

### 1.2 핵심 가치
- **맞춤성**: 체형과 목표에 따라 무리 없는 운동만 골라 추천 (예: 과체중+하체 → 무릎 부담 적은 운동)
- **접근성**: 교실 의자에 앉아 10분 안에 할 수 있는 틈새 스트레칭
- **지속성**: 출석·경험치·캐릭터 진화로 매일 다시 열게 만드는 게임형 동기부여

### 1.3 타겟 & 사용 맥락
하루 종일 앉아 있는 중·고등학생. 쉬는 시간, 자습 시간, 방과 후 짧은 시간에 스마트폰이나 교실 PC로 접속. 이 타겟 특성상 **거북목·굽은 등·손목 피로** 등 좌식 생활 문제 해결이 강한 진입점.

### 1.4 플랫폼 전략
1단계 **웹(반응형)** 으로 전체 기능 완성 → 2단계 동일 코드베이스를 **모바일 앱**으로 포장(PWA 또는 React Native/WebView). 자세한 전환·호스팅 전략은 9장 참조.

## 3. 맞춤 추천 알고리즘 상세

추천은 **3개의 입력**을 조합해 결정된다: `체형(BMI 분류)` × `목표` × `부위`. 아래 규칙 테이블을 코드로 옮기면 그대로 추천 엔진이 된다.

### 3.1 BMI 계산 및 분류 (F1)

```
BMI = 몸무게(kg) / (키(m) × 키(m))
```

분류 기준 (대한비만학회 / 아시아-태평양 기준):

| 분류 | BMI 범위 | 내부 코드 |
|------|----------|-----------|
| 저체중 | < 18.5 | `UNDER` |
| 정상 | 18.5 ~ 22.9 | `NORMAL` |
| 과체중 | 23.0 ~ 24.9 | `OVER` |
| 비만 | ≥ 25.0 | `OBESE` |

> ⚠️ 성장기 청소년은 성인 BMI 기준을 그대로 적용하기 어렵다. **실서비스에서는 연령·성별 소아청소년 BMI 백분위수(성장도표)** 를 쓰는 것이 정확하다. MVP에서는 성인 기준으로 단순 계산하되, 문구에 "참고용" 표기를 권장. (11장 안전 참조)

### 3.2 목표 (F2)

| 목표 | 코드 | 운동 성향 |
|------|------|-----------|
| 근육량 증가 | `MUSCLE` | 근력·저항 운동 중심 |
| 체지방 감소 | `FAT_LOSS` | 유산소·전신·저충격 중심 |
| 체력 증진 | `STAMINA` | 유산소+코어, 중강도 지속 |

### 3.3 부위 (F3)

`상체(UPPER)`, `하체(LOWER)`, `코어/복부(CORE)`, `전신(FULL)` — 사용자가 집중하고 싶은 부위 선택.

### 3.4 추천 매칭 규칙 (핵심 로직)

체형에 따라 **안전 필터**가 먼저 작동한 뒤, 목표·부위로 운동을 고른다.

**안전 필터 규칙**
- `OVER` / `OBESE` + `LOWER` → 무릎·관절 고충격 운동 제외 (점프·달리기·풀스쿼트 배제). **저충격(low-impact)** 대안으로 대체.
- `UNDER` → 과도한 유산소보다 **근력·영양 병행** 메시지 우선.
- 모든 조합 → 초보자용 **저난이도 버전**을 기본 노출, "레벨업 시 상위 동작 해금".

**대표 매칭 예시**

| 체형 | 목표 | 부위 | 추천 운동 (예) | 이유 |
|------|------|------|----------------|------|
| 과체중/비만 | 체지방 감소 | 하체 | 수영, 가벼운 스쿼트(의자 스쿼트), 실내 자전거, 빠르게 걷기 | 무릎 부담 최소화한 저충격 유산소·근력 |
| 저체중 | 근육량 증가 | 상체 | 푸시업(무릎 대고 시작), 밴드 로우, 덤벨 컬 | 저항 운동으로 근합성 자극 |
| 정상 | 체력 증진 | 전신 | 버피(저강도), 마운틴 클라이머, 점핑잭 | 심폐지구력 + 전신 순환 |
| 정상 | 근육량 증가 | 코어 | 플랭크, 데드버그, 레그레이즈 | 코어 안정성·근지구력 |
| 과체중 | 체력 증진 | 전신 | 빠르게 걷기, 수중 걷기, 스텝박스(낮은 높이) | 관절 보호하며 심박 상승 |
| 저체중 | 체력 증진 | 하체 | 바디웨이트 스쿼트, 런지(보조), 카프레이즈 | 하체 근력 기반 체력 |

### 3.5 추천 엔진 의사코드 (v0.2 — 점수 기반 랭킹)

하드 필터로 3개만 남기던 방식에서, **안전 필터를 통과한 전체 운동을 적합도 점수로 정렬**하는 방식으로 변경했다. 홈에서는 상위 3개를 기본 노출하고, **`더보기`로 최대 9개까지** 펼쳐 볼 수 있다(가장 잘 맞는 순).

```pseudo
function recommend(bmiClass, goal, focusParts):
    if bmiClass == UNDER and goal == FAT_LOSS:
        goal = STAMINA                    # 저체중은 과한 감량 유도 금지
    pool = ALL_EXERCISES
    if bmiClass in [OVER, OBESE]:
        pool = pool.filter(e => e.impact != "HIGH")   # 안전 필터: 고충격 제외

    score(e):
        s = 0
        if goal in e.goals:            s += 5   # 목표 일치
        if e.part in focusParts:       s += 3   # 집중 부위 일치
        if e.part == "FULL":           s += 1   # 전신은 두루 적합
        if e.impact == "LOW":          s += 0.5 # 저충격 가산
        if e.minLevel > user.level:    s -= 2   # 레벨 부족 운동은 뒤로
        return s

    return pool.sortByDesc(score)         # 홈: 기본 3개 / 더보기 시 9개
```

> `더보기`/`접기`는 상태 플래그(`recExpanded`) 하나로 토글. 과체중·비만은 고충격 운동이 제외되므로 최대 노출 수가 8개 이하가 될 수 있다(안전 우선).

## 4. 10분 틈새 스트레칭 (F4)

교실 의자에 앉아 할 수 있는 **좌식 스트레칭 루틴**. 부위별로 묶고, 한 세트를 10분 안에 끝낼 수 있게 구성.

| 루틴 | 부위 | 동작 예시 | 소요 |
|------|------|-----------|------|
| 거북목 교정 | 목/어깨 | 턱 당기기, 목 옆으로 늘이기, 어깨 으쓱 | 3분 |
| 굽은 등 펴기 | 등/가슴 | 의자 등받이 활용 흉추 신전, 날개뼈 모으기 | 3분 |
| 손목·팔 이완 | 손목/팔 | 손목 털기, 손목 돌리기, 손가락 스트레칭 | 2분 |
| 하체 순환 | 다리 | 앉아서 무릎 펴기, 발목 펌프, 종아리 늘이기 | 2분 |

각 동작은 클립 + 반복 횟수/시간 안내. 루틴 완료 시 EXP 지급(5장 연동). "쉬는 시간 10분" 프리셋으로 원터치 시작 제공.

## 5. 게이미피케이션 — 출석·경험치·캐릭터 (F5)

### 5.1 경험치(EXP) 지급 규칙

| 행동 | EXP |
|------|-----|
| 추천 운동 1개 완료 | +10 |
| 틈새 스트레칭 루틴 완료 | +15 |
| 일일 목표(3개 완료) 달성 | +20 보너스 |
| 출석(하루 첫 접속+운동) | +5 |
| 연속 출석 스트릭 보너스 | +연속일수 (최대 +30) |

### 5.2 레벨 공식

```
Lv N 도달 필요 누적 EXP = 50 × N × (N + 1) / 2
# Lv1→2: 100 / Lv2→3: 150 / Lv3→4: 200 ... 완만한 곡선
```

### 5.3 캐릭터 진화 단계

레벨 구간마다 캐릭터가 성장·진화한다. 학생 친화적인 마스코트(예: 알 → 새싹 캐릭터 → 성장형).

| 단계 | 레벨 | 모습(예시) |
|------|------|-----------|
| 1. 알 | Lv 1~4 | 웅크린 알 |
| 2. 부화 | Lv 5~9 | 갓 깨어난 새싹 캐릭터 |
| 3. 성장 | Lv 10~19 | 운동복 입은 활발한 캐릭터 |
| 4. 각성 | Lv 20~34 | 근육/체력 강조된 진화형 |
| 5. 마스터 | Lv 35+ | 반짝이는 최종 진화형 |

진화 순간에는 **연출(애니메이션 + 축하 모달)** 로 성취감 강화. 캐릭터는 홈 화면 중앙에 항상 노출.

### 5.4 출석체크
달력 위젯으로 이번 달 출석 표시. 연속 출석(스트릭) 강조. 하루라도 빠지면 스트릭 리셋(단, 주 1회 "방어권" 아이템으로 보호 가능 — v1.1 확장 아이디어).

### 5.5 EXP 획득 제한 — 쿨다운 규칙 (v0.2 신규)

같은 운동을 반복 클릭해 EXP를 무한정 버는 어뷰징을 막기 위해 **운동별 쿨다운**을 둔다.

- **규칙**: 한 운동을 완료하면 그 운동은 **쿨다운 시간 동안 EXP 재획득 불가**. 다른 운동은 영향받지 않으므로, 여러 운동을 하면 계속 EXP를 얻을 수 있다.
- **쿨다운 값**: 운영 기준 **3시간**(권장). 코드 상단 `COOLDOWN_MS` 한 줄로 조절. *현재 프로토타입은 테스트 편의를 위해 **1분**으로 설정되어 있음 → 운영 전환 시 `3*60*60*1000`으로 변경.*
- **완료/미완료 버튼 구분**:
  - 미완료: `✅ 완료했어요! (+10 EXP)` — 클릭 가능(초록 채움).
  - 완료(쿨다운 중): `✅ 완료함 · N분 후 다시` — **클릭 비활성화**(연초록 테두리), 남은 시간 **매초 실시간 표시**. 쿨다운이 끝나면 자동 재활성화.
  - 홈·스트레칭 목록 카드에도 `✅ 완료 · N분 후` 배지와 체크(✓) 표시로 한눈에 구분.

> EXP 값: 추천 운동 +10 / 틈새 스트레칭 +15 (5.1과 동일). 완료 시 컨페티·레벨업 연출 발생.

## 7. 화면 흐름(IA) & 유저 플로우

### 7.1 화면 목록
1. **온보딩** — 키·몸무게·목표·집중 부위 입력 → BMI 결과 표시
2. **홈** — 캐릭터(중앙), 오늘의 추천(기본 3개·더보기 최대 9개), 출석/스트릭, EXP 바
3. **틈새 스트레칭** — "쉬는 시간 10분" 프리셋, 부위별 루틴
4. **운동 상세/플레이어** — 유튜브 영상(or 애니메이션 폴백), 설명, "완료" 버튼(쿨다운 표시)
5. **성장** — 레벨/캐릭터 진화 도감, 통계, **내 정보 수정**(닉네임·키·몸무게·목표·부위)

### 7.2 핵심 유저 플로우

```
[온보딩] 키·몸무게·목표·부위 입력
   → BMI 계산·분류
   → [홈] 맞춤 추천 3개 노출 + 캐릭터
   → 운동 선택 → [플레이어] 클립 보며 수행 → "완료"
   → EXP 지급 → (레벨업 시) 캐릭터 진화 연출
   → 출석·스트릭 갱신 → 홈 복귀
```

### 8.4 ⚠️ 임베드 재생 조건 (중요)
유튜브 플레이어는 **페이지 출처(origin)가 있는 http(s) 환경에서만 임베드 재생**된다. HTML 파일을 `file://`로 직접 열면 오류(153 등)로 재생되지 않는다 → **반드시 웹 서버(로컬 또는 원격)로 열어야 함**(9.3 참조).

### 8.5 저작권
현재는 프로토타입 검증용 임베드. **정식 서비스 배포 전 영상 소유자(자생한방병원) 동의/제휴 또는 자체 제작 클립으로 대체** 필요. (11장 저작권 참조)

## 9. 기술 스택 & 웹 → 앱 전환 전략

### 9.1 웹 MVP
- 프론트: HTML/CSS/JS 또는 React. 상태 저장은 localStorage.
- 서버리스로 시작 가능(정적 호스팅). Netlify·GitHub Pages·Cloudflare Pages 등 무료 정적 호스팅에 그대로 배포 가능.

### 9.2 웹 → 앱 전환 3가지 경로

| 방식 | 장점 | 단점 | 추천도 |
|------|------|------|:--:|
| **PWA** (웹앱 설치) | 코드 재사용 100%, 홈화면 추가·오프라인·푸시 일부 | 앱스토어 미노출, iOS 푸시 제약 | ★ 빠른 1차 |
| **WebView 래핑**(Capacitor 등) | 웹 코드 그대로 앱 패키징, 스토어 등록 | 네이티브 성능·UX 한계 | ★ 실용적 |
| **React Native 재작성** | 진짜 네이티브 UX·성능 | 재개발 비용, 별도 코드 | 장기 |

권장: **웹으로 완성 → PWA로 즉시 "앱처럼" → 반응 좋으면 Capacitor로 스토어 등록**. React로 짜두면 이후 React Native 전환 시 로직 재사용에 유리.

### 9.3 실행/호스팅 전략 — 로컬(현재) → 원격(예정)

유튜브 임베드가 http(s) 출처를 요구하므로(8.4), 앱은 **웹 서버로 서빙**해야 한다. 단계별 계획:

| 단계 | 방식 | 용도 | 상태 |
|------|------|------|------|
| 1 | **로컬 서버** (`python3 -m http.server`) → `http://localhost:8000/` | 개발 중 내 PC에서 확인 | **현재** |
| 2 | **원격 서버 배포** — **GitHub Pages** (선택 확정) → 공유 URL | 공유·실사용·모바일 접속·PWA 설치 | **예정** |

- 프로토타입 파일명을 서버 루트에서 바로 열리도록 **`index.html`** 로 정리함.
- **로컬 실행(현재)**:
  ```bash
  cd "/Users/sonypark/dev-study/body-fit-app"
  python3 -m http.server 8000
  # 브라우저: http://localhost:8000/
  ```
- **원격 배포(예정) — GitHub Pages 채택**: 순수 정적 파일(`index.html`) 하나이므로 GitHub 저장소에 올려 Pages를 켜면 즉시 공유 URL을 얻는다. HTTPS 기본 제공이라 유튜브 임베드가 정상 재생되고, PWA 설치·모바일 접속도 가능해진다. **push만 하면 자동 재배포**되므로 계속 수정하는 프로젝트에 적합하다.

#### 9.3.1 GitHub Pages 배포 절차

1. **저장소 생성(웹)**: github.com → New repository → 이름 `body-fit-app` → **Public** → (README·.gitignore 옵션 체크 해제, 이미 로컬에 있음) → Create.
2. **코드 업로드(터미널)**:
   ```bash
   cd "/Users/sonypark/dev-study/body-fit-app"
   git init
   git add .
   git commit -m "무브핏 프로토타입 배포"
   git branch -M main
   git remote add origin https://github.com/<사용자명>/body-fit-app.git
   git push -u origin main
   ```
   > push 시 GitHub 로그인(브라우저 인증 또는 Personal Access Token)이 필요하다. *(터미널이 번거로우면 저장소 화면의 "uploading an existing file"로 파일을 드래그&드롭 커밋해도 동일)*
3. **Pages 활성화**: 저장소 → Settings → Pages → Source: *Deploy from a branch* → Branch: **main / (root)** → Save.
4. **접속**: 1~2분 뒤 `https://<사용자명>.github.io/body-fit-app/` (루트의 `index.html` 자동 서빙).

> ⚠️ **공개 범위**: Public 저장소는 `기획설계서.md`도 함께 공개된다. 앱만 공개하려면 저장소에 `index.html`만 올릴 것. (비공개 저장소로 Pages를 쓰려면 유료 플랜 필요)

- **대안(참고)**: 빠른 일회성 링크는 Netlify Drop(폴더 드래그&드롭), 그 외 Cloudflare Pages·Vercel도 무료·HTTPS·정적 배포 지원. 커스텀 도메인·HTTPS는 모두 무료. (무료 티어 세부 한도는 각 서비스 최신 정책 확인 권장)
- **커스텀 도메인(선택)**: 저장소 Settings → Pages → Custom domain에 도메인 연결 가능(무료 HTTPS 자동 발급).

## 11. 안전 · 정책 고려사항

- **건강 정보 면책**: BMI·추천은 참고용이며 의학적 진단이 아님을 명시. 통증·질환 시 전문가 상담 권고 문구.
- **청소년 대상**: 성장기 특성상 성인 BMI 기준의 한계 안내. 과한 체중 감량·외모 압박을 유발하지 않는 **긍정적·건강 중심 카피**. 체중/체형 수치를 부정적으로 강조하지 않기.
- **개인정보**: 키·몸무게는 민감할 수 있으므로 서버 저장 최소화(웹 MVP는 로컬 저장). 실명·연락처 등 불필요한 개인정보 수집 지양.
- **저작권**: 영상은 직접 촬영 또는 라이선스 확보분만 사용.
