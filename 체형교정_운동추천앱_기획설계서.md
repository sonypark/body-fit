## 1. 개요

### 1.1 한 줄 정의
키·몸무게와 목표를 입력하면 **체형(BMI)·목표·운동 부위**에 맞는 운동/스트레칭을 짧은 클립과 함께 추천하고, 완료할 때마다 **경험치와 캐릭터 성장**으로 꾸준함을 보상하는 학생용 운동 코치 앱.

### 1.2 핵심 가치
- **맞춤성**: 체형과 목표에 따라 무리 없는 운동만 골라 추천 (예: 과체중+하체 → 무릎 부담 적은 운동)
- **접근성**: 교실 의자에 앉아 10분 안에 할 수 있는 틈새 스트레칭
- **지속성**: 출석·경험치·캐릭터 진화로 매일 다시 열게 만드는 게임형 동기부여

### 1.3 타겟 & 사용 맥락
하루 종일 앉아 있는 중·고등학생. 쉬는 시간, 자습 시간, 방과 후 짧은 시간에 스마트폰이나 교실 PC로 접속. 이 타겟 특성상 **거북목·굽은 등·손목 피로** 등 좌식 생활 문제 해결이 강한 진입점.

### 1.4 플랫폼 전략
1단계 **웹(반응형)** 으로 전체 기능 완성 → 2단계 동일 코드베이스를 **모바일 앱**으로 포장(PWA 또는 React Native/WebView). 자세한 전환·호스팅 전략은 9장 참조.

## 3. 맞춤 추천 알고리즘 상세

추천은 **3개의 입력**을 조합해 결정된다: `체형(BMI 분류)` × `목표` × `부위`. 아래 규칙 테이블을 코드로 옮기면 그대로 추천 엔진이 된다.

### 3.1 BMI 계산 및 분류 (F1)

```
BMI = 몸무게(kg) / (키(m) × 키(m))
```

분류 기준 (대한비만학회 / 아시아-태평양 기준):

| 분류 | BMI 범위 | 내부 코드 |
|------|----------|-----------|
| 저체중 | < 18.5 | `UNDER` |
| 정상 | 18.5 ~ 22.9 | `NORMAL` |
| 과체중 | 23.0 ~ 24.9 | `OVER` |
| 비만 | ≥ 25.0 | `OBESE` |

> ⚠️ 성장기 청소년은 성인 BMI 기준을 그대로 적용하기 어렵다. **실서비스에서는 연령·성별 소아청소년 BMI 백분위수(성장도표)** 를 쓰는 것이 정확하다. MVP에서는 성인 기준으로 단순 계산하되, 문구에 "참고용" 표기를 권장. (11장 안전 참조)

### 3.2 목표 (F2)

| 목표 | 코드 | 운동 성향 |
|------|------|-----------|
| 근육량 증가 | `MUSCLE` | 근력·저항 운동 중심 |
| 체지방 감소 | `FAT_LOSS` | 유산소·전신·저충격 중심 |
| 체력 증진 | `STAMINA` | 유산소+코어, 중강도 지속 |

### 3.3 부위 (F3)

`상체(UPPER)`, `하체(LOWER)`, `코어/복부(CORE)`, `전신(FULL)` — 사용자가 집중하고 싶은 부위 선택.

### 3.4 추천 매칭 규칙 (핵심 로직)

체형에 따라 **안전 필터**가 먼저 작동한 뒤, 목표·부위로 운동을 고른다.

**안전 필터 규칙**
- `OVER` / `OBESE` + `LOWER` → 무릎·관절 고충격 운동 제외 (점프·달리기·풀스쿼트 배제). **저충격(low-impact)** 대안으로 대체.
- `UNDER` → 과도한 유산소보다 **근력·영양 병행** 메시지 우선.
- 모든 조합 → 초보자용 **저난이도 버전**을 기본 노출, "레벨업 시 상위 동작 해금".

**대표 매칭 예시** *(규칙 설명용 개념 예시 — 실제 수록 콘텐츠는 8장 참조)*

| 체형 | 목표 | 부위 | 추천 운동 (예) | 이유 |
|------|------|------|----------------|------|
| 과체중/비만 | 체지방 감소 | 하체 | 수영, 가벼운 스쿼트(의자 스쿼트), 실내 자전거, 빠르게 걷기 | 무릎 부담 최소화한 저충격 유산소·근력 |
| 저체중 | 근육량 증가 | 상체 | 푸시업(무릎 대고 시작), 밴드 로우, 덤벨 컬 | 저항 운동으로 근합성 자극 |
| 정상 | 체력 증진 | 전신 | 버피(저강도), 마운틴 클라이머, 점핑잭 | 심폐지구력 + 전신 순환 |
| 정상 | 근육량 증가 | 코어 | 플랭크, 데드버그, 레그레이즈 | 코어 안정성·근지구력 |
| 과체중 | 체력 증진 | 전신 | 빠르게 걷기, 수중 걷기, 스텝박스(낮은 높이) | 관절 보호하며 심박 상승 |
| 저체중 | 체력 증진 | 하체 | 바디웨이트 스쿼트, 런지(보조), 카프레이즈 | 하체 근력 기반 체력 |

### 3.5 추천 엔진 의사코드 (v0.4 — 부위 필터 + 점수 랭킹)

v0.2의 "부위 일치 가산점" 방식에서, **선택한 집중 부위 카테고리의 운동만 노출하는 하드 필터** 방식으로 변경했다. 상체만 선택하면 상체 영상만, 상체+전신을 선택하면 두 카테고리 영상이 모두 나온다. 홈에서는 상위 3개를 기본 노출하고, `더보기`로 **선택 부위의 전체 목록**을 펼칠 수 있다(9개 상한 제거).

```pseudo
function recommend(bmiClass, goal, focusParts):
    if bmiClass == UNDER and goal == FAT_LOSS:
        goal = STAMINA                    # 저체중은 과한 감량 유도 금지
    pool = ALL_EXERCISES.filter(e => e.part in focusParts)  # 선택 부위 카테고리만
    if bmiClass in [OVER, OBESE]:
        pool = pool.filter(e => e.impact != "HIGH")   # 안전 필터: 고충격 제외

    score(e):
        s = 0
        if goal in e.goals:            s += 5    # 목표 일치
        if e.impact == "LOW":          s += 0.5  # 저충격 가산
        if e.minLevel > user.level:    s -= 100  # 레벨 잠금 운동은 맨 뒤(🔒 표시)
        return s

    return pool.sortByDesc(score)   # 동점 시 카탈로그 순서(=카테고리 내 조회수 순) 유지
```

> `더보기`/`접기`는 상태 플래그(`recExpanded`) 토글이며, **홈 화면 진입 시 항상 접힌 상태로 초기화**된다. 레벨 미달 운동은 목록 맨 뒤에 🔒 잠금 카드로 표시되고 클릭 시 해금 레벨을 안내한다.

## 5. 게이미피케이션 — 출석·경험치·캐릭터 (F5)

### 5.1 경험치(EXP) 지급 규칙

| 행동 | EXP |
|------|-----|
| 추천 운동 1개 완료 | +10 |
| 틈새 스트레칭 루틴 완료 | +15 |
| 일일 목표(3개 완료) 달성 | +20 보너스 |
| 출석(하루 첫 접속+운동) | +5 |
| 연속 출석 스트릭 보너스 | +연속일수 (최대 +30) |

### 5.2 레벨 공식 (v0.4 — 3레벨 체계로 단순화)

레벨 = 진화 단계와 1:1 매칭되는 3레벨 체계.

```
Lv 1 (1단계): 시작 ~ 99 EXP
Lv 2 (2단계): 100 EXP 도달 시 1차 진화
Lv 3 (3단계): 300 EXP 도달 시 최종 진화 (이후 MAX 표시)
```

### 5.3 캐릭터 — 오리지널 속성 스타터 3종 × 3단 진화 (v0.3 전면 개편)

온보딩에서 **불/물/풀 속성 파트너 중 1종을 선택**하고, 운동할수록 3단계까지 진화한다.
(포켓몬 등 기존 IP는 **저작권 문제로 사용 불가** → 속성 스타터 컨셉만 차용한 **완전 오리지널 SVG 캐릭터**로 직접 디자인. 실제 포켓몬과 실루엣·색·모티프가 겹치지 않도록 유지할 것.)

| 속성 | 1단계 (Lv1) | 2단계 (Lv2) | 3단계 (Lv3) |
|------|------------|------------|------------|
| 🔥 불 | 카이엔 | 파이락스 | 볼자로스 |
| 💧 물 | 미스트랄 | 티드락 | 리비아론 |
| 🌱 풀 | 베인 | 브램블록 | 베르단토르 |

- SVG 인라인 렌더링: 그라데이션 음영·하이라이트·외곽선, 속성별 idle 파티클(불꽃/물방울/잎, 단계가 오를수록 증가), 눈 깜빡임·호흡 애니메이션, 3단계는 날카로운 눈매의 당당한 포즈.
- **진화 연출 3막**: 실루엣 → 흰 빛 플래시 → 새 모습 공개 (+ 컨페티).
- 홈 캐릭터 **탭 리액션**: 점프 + 랜덤 응원 멘트.
- 구버전 데이터(캐릭터 미선택) 사용자는 접속 시 파트너 선택 시트가 1회 표시됨(마이그레이션).

### 5.4 출석체크 — ✅ 달력 구현 완료
홈의 연속 출석 카드(최근 7일 스트립)를 누르면 **월별 출석 달력 시트**가 열린다(월 이동, 출석일·오늘 표시, 월 출석 수 요약). 스트릭 방어권 아이템은 v1.1 확장 아이디어로 유지.

### 5.5 EXP 획득 제한 — 쿨다운 규칙 (v0.2 신규)

같은 운동을 반복 클릭해 EXP를 무한정 버는 어뷰징을 막기 위해 **운동별 쿨다운**을 둔다.

- **규칙**: 한 운동을 완료하면 그 운동은 **쿨다운 시간 동안 EXP 재획득 불가**. 다른 운동은 영향받지 않으므로, 여러 운동을 하면 계속 EXP를 얻을 수 있다.
- **쿨다운 값**: **3시간** — ✅ 운영값 적용 완료 (`COOLDOWN_MS = 3*60*60*1000`).
- **완료/미완료 버튼 구분**:
  - 미완료: `✅ 완료했어요! (+10 EXP)` — 클릭 가능(초록 채움).
  - 완료(쿨다운 중): `✅ 완료함 · N분 후 다시` — **클릭 비활성화**(연초록 테두리), 남은 시간 **매초 실시간 표시**. 쿨다운이 끝나면 자동 재활성화.
  - 홈·스트레칭 목록 카드에도 `✅ 완료 · N분 후` 배지와 체크(✓) 표시로 한눈에 구분.

> EXP 값: 추천 운동 +10 / 틈새 스트레칭 +15 (5.1과 동일). 완료 시 컨페티·레벨업 연출 발생.

## 7. 화면 흐름(IA) & 유저 플로우

### 7.1 화면 목록
1. **온보딩(6스텝)** — 환영 → **파트너 캐릭터 선택(3종 중 1택)** → 키·몸무게 → BMI 결과 → 목표 → 집중 부위. 뒤로가기(`‹`)·입력 검증 지원
2. **홈** — 캐릭터(중앙, 탭 리액션), 오늘의 추천(기본 3개·더보기로 선택 부위 전체), 출석/스트릭(탭 시 월 달력), EXP 바
3. **틈새 스트레칭** — "쉬는 시간 10분 풀세트"(핵심 4종 순차 진행), 부위별 6그룹 32개
4. **운동 상세/플레이어** — 유튜브 영상(or 애니메이션 폴백), 설명·"이렇게 해요", "완료" 버튼(쿨다운 표시)
5. **성장** — 캐릭터 진화 도감(3단계), 내 BMI, 통계, **내 정보 수정**, 초기화(커스텀 확인 시트)

### 7.2 핵심 유저 플로우

```
[온보딩] 파트너 선택 → 키·몸무게 → BMI → 목표·부위 입력
   → [홈] 선택 부위 맞춤 추천 3개 + 캐릭터
   → 운동 선택 → [플레이어] 클립 보며 수행 → "완료"
   → EXP 지급 → (100/300 EXP 도달 시) 진화 연출 3막
   → 출석·스트릭 갱신 → 홈 복귀
```

## 8. 영상 콘텐츠 처리 (v0.4 — 콘텐츠 소스 2원화, 총 74개)

### 8.1 콘텐츠 소스

| 탭 | 소스 | 개수 | 구성 기준 |
|----|------|:--:|-----------|
| 🏠 홈 "오늘의 맞춤 추천" | **Nobadaddiction** 채널 쇼츠 (`@Nobadaddiction`) | 42 | 채널 **인기순** 정렬에서 운동 가이드 영상만 선별(레시피·밈·동일 제목 재업로드 제외). 카테고리별: 상체 4 / 하체 5 / 코어 4 / 전신 29 — 각 카테고리 내 조회수 순 |
| 🪑 스트레칭 탭 | 자생한방병원 **"숏트레칭"** 재생목록 (`PLeOfuaJ_0peAhL_gX7pPeWKyZngE4xfM6`) | 32 | 재생목록 전체 수록, 부위별 6그룹(4장 참조) |

- 대표 인기 영상: 살빠지는 간단 운동 6가지(503만), 엉덩이 집중 홈트(243만), 윗배 vs 아랫배(152만), 물구나무 서기(103만) 등.
- 각 항목의 **이름·설명·"👀 이렇게 해요"는 영상 제목·설명·자막 내용 기준**으로 작성 (앱 텍스트 ↔ 영상 내용 일치 원칙).
- 운동별 메타데이터(`part`/`goals`/`impact`/`minLevel`)를 부여해 추천 엔진(3.5)과 연동 — 고강도 HIIT류는 `HIGH`로 지정되어 과체중/비만 사용자에게 안전 필터로 제외됨.
- 전체 매칭은 `index.html`의 `EX`(42개)·`STRETCH`(32개) 배열이 단일 소스(Source of Truth).

### 8.3 재생 방식
- 목록 카드 썸네일: 유튜브 썸네일(`i.ytimg.com/vi/<ID>/hqdefault.jpg`) 사용.
- 상세 플레이어: **세로형(9:16) 유튜브 임베드**(`youtube.com/embed/<ID>`)로 앱 안에서 재생.
- `vid`가 없는 항목은 기존 **애니메이션 플레이스홀더**로 폴백(분기 처리 유지).
- 임베드가 막히는 환경을 대비해 **"▶ 유튜브에서 보기"** 링크(`youtube.com/shorts/<ID>`)를 항상 병행 노출.

### 8.4 ⚠️ 임베드 재생 조건 (중요)
유튜브 플레이어는 **페이지 출처(origin)가 있는 http(s) 환경에서만 임베드 재생**된다. HTML 파일을 `file://`로 직접 열면 오류(153 등)로 재생되지 않는다 → **반드시 웹 서버(로컬 또는 원격)로 열어야 함**(9.3 참조).

### 8.5 저작권
현재는 프로토타입 검증용 임베드. **정식 서비스 배포 전 영상 소유자(자생한방병원, Nobadaddiction) 동의/제휴 또는 자체 제작 클립으로 대체** 필요. (11장 저작권 참조)
캐릭터는 기존 IP(포켓몬 등) 사용이 저작권 침해라 **오리지널 디자인으로 확정**함(5.3).

## 9. 기술 스택 & 웹 → 앱 전환 전략

### 9.1 웹 MVP
- 프론트: HTML/CSS/JS 또는 React. 상태 저장은 localStorage.
- 서버리스로 시작 가능(정적 호스팅). Netlify·GitHub Pages·Cloudflare Pages 등 무료 정적 호스팅에 그대로 배포 가능.

### 9.2 웹 → 앱 전환 3가지 경로

| 방식 | 장점 | 단점 | 추천도 |
|------|------|------|:--:|
| **PWA** (웹앱 설치) | 코드 재사용 100%, 홈화면 추가·오프라인·푸시 일부 | 앱스토어 미노출, iOS 푸시 제약 | ★ 빠른 1차 |
| **WebView 래핑**(Capacitor 등) | 웹 코드 그대로 앱 패키징, 스토어 등록 | 네이티브 성능·UX 한계 | ★ 실용적 |
| **React Native 재작성** | 진짜 네이티브 UX·성능 | 재개발 비용, 별도 코드 | 장기 |

권장: **웹으로 완성 → PWA로 즉시 "앱처럼" → 반응 좋으면 Capacitor로 스토어 등록**. React로 짜두면 이후 React Native 전환 시 로직 재사용에 유리.

### 9.3 실행/호스팅 전략 — 로컬(현재) → 원격(예정)

유튜브 임베드가 http(s) 출처를 요구하므로(8.4), 앱은 **웹 서버로 서빙**해야 한다. 단계별 계획:

| 단계 | 방식 | 용도 | 상태 |
|------|------|------|------|
| 1 | **로컬 서버** (`python3 -m http.server`) → `http://localhost:8000/` | 개발 중 내 PC에서 확인 | 사용 중 |
| 2 | **GitHub Pages** — 저장소 `github.com/sonypark/body-fit` (main 브랜치) → `https://sonypark.github.io/body-fit/` | 공유·실사용·모바일 접속·PWA 설치 | ✅ **배포 완료** (push 시 자동 재배포) |

> 저장소에는 `index.html`·`README.md`·기획설계서만 추적하고, 도구 내부 상태 폴더(`.omc/`)와 `.DS_Store`는 `.gitignore`로 제외한다.

- 프로토타입 파일명을 서버 루트에서 바로 열리도록 **`index.html`** 로 정리함.
- **로컬 실행(현재)**:
  ```bash
  cd "/Users/sonypark/dev-study/body-fit-app"
  python3 -m http.server 8000
  # 브라우저: http://localhost:8000/
  ```
- **원격 배포(✅ 완료) — GitHub Pages**: 순수 정적 파일(`index.html`)이므로 저장소에 올려 Pages를 켜는 것만으로 공유 URL 확보. HTTPS 기본 제공이라 유튜브 임베드가 정상 재생되고, PWA 설치·모바일 접속도 가능하다. **main에 push만 하면 자동 재배포**된다.

#### 9.3.1 배포 정보 (완료된 설정)

- 저장소: `git@github.com:sonypark/body-fit.git` (Public, 기본 브랜치 `main`)
- Pages: Settings → Pages → *Deploy from a branch* → **main / (root)**
- 접속: `https://sonypark.github.io/body-fit/` (루트의 `index.html` 자동 서빙)
- 배포 방법: `git push` (1~2분 내 자동 반영)

> ⚠️ **공개 범위**: Public 저장소라 `기획설계서.md`·`README.md`도 함께 공개된다. 앱만 공개하려면 별도 저장소에 `index.html`만 분리할 것.

- **대안(참고)**: 빠른 일회성 링크는 Netlify Drop(폴더 드래그&드롭), 그 외 Cloudflare Pages·Vercel도 무료·HTTPS·정적 배포 지원. 커스텀 도메인·HTTPS는 모두 무료. (무료 티어 세부 한도는 각 서비스 최신 정책 확인 권장)
- **커스텀 도메인(선택)**: 저장소 Settings → Pages → Custom domain에 도메인 연결 가능(무료 HTTPS 자동 발급).

## 11. 안전 · 정책 고려사항

- **건강 정보 면책**: BMI·추천은 참고용이며 의학적 진단이 아님을 명시. 통증·질환 시 전문가 상담 권고 문구.
- **청소년 대상**: 성장기 특성상 성인 BMI 기준의 한계 안내. 과한 체중 감량·외모 압박을 유발하지 않는 **긍정적·건강 중심 카피**. 체중/체형 수치를 부정적으로 강조하지 않기.
- **개인정보**: 키·몸무게는 민감할 수 있으므로 서버 저장 최소화(웹 MVP는 로컬 저장). 실명·연락처 등 불필요한 개인정보 수집 지양.
- **저작권**: 영상은 직접 촬영 또는 라이선스 확보분만 사용.

- **UX/버그 수정 일괄**: 온보딩 뒤로가기·입력 검증, 출석 날짜 UTC→로컬 버그 수정, 쿨다운 운영값(3시간) 적용, 반응형 폭(`min(390px,100vw)`), 확대 차단 해제(접근성), 썸네일 로드 실패 폴백, `confirm()`→커스텀 초기화 시트, 레벨 잠금 카드(🔒) 시각화, `prefers-reduced-motion` 지원.
